## <strong>📌 프로젝트 소개</strong>

- <strong>DDui-DDui-Ride</strong>는 Uber와 유사한 차량 호출 서비스를 구현한 풀스택 웹 애플리케이션입니다. Google Maps API를 활용하여 실시간 경로 계산 및 시각화를 제공하며, Stripe를 통한 안전한 결제 처리를 지원합니다.
- <strong>개발 기간</strong>: 2024년 (3일)

## <strong>🎯 프로젝트 목표</strong>


---

- <strong>Google Maps API 심층 학습</strong>: Mapping library의 다양한 기능(Places API, Directions API, Custom Markers 등)을 실전에서 활용
- <strong>한국 환경 최적화</strong>: 해외 튜토리얼 기반 프로젝트를 한국 환경에 맞게 커스터마이징
- <strong>풀스택 개발 경험</strong>: 프론트엔드부터 결제 시스템까지 전체 플로우 구현

## <strong>✨ 주요 기능</strong>


---


<strong>🗺️ 지도 및 경로 기능</strong>

- <strong>실시간 장소 검색</strong>: Google Places Autocomplete API를 활용한 출발지/도착지 검색
- <strong>경로 시각화</strong>: DirectionsRenderer를 통한 최적 경로 표시
- <strong>커스텀 마커</strong>: 출발지와 도착지를 구분하는 맞춤형 마커 및 레이블
- <strong>동적 지도 이동</strong>: 장소 선택 시 자동으로 해당 위치로 지도 중심 이동

<strong>🚘 차량 선택 시스템</strong>

- <strong>다양한 차량 옵션</strong>: Uber X, Comfort, XL, Pet, Black 등 5가지 차량 타입
- <strong>실시간 요금 계산</strong>: 거리 기반 동적 요금 산정
- <strong>차량별 상세 정보</strong>: 좌석 수, 특징, 가격 정보 제공

<strong>💳 결제 시스템</strong>

- <strong>Stripe 통합</strong>: 안전한 카드 결제 처리
- <strong>결제 확인 페이지</strong>: 결제 완료 후 예약 확인 화면

<strong>🔐 사용자 인증</strong>

- <strong>Clerk 인증 시스템</strong>: 안전한 회원가입 및 로그인
- <strong>보호된 라우트</strong>: 인증된 사용자만 서비스 이용 가능

## <strong>🛠️ 기술 스택</strong>


---


### <strong>Frontend</strong>

- <strong>Framework</strong>: Next.js 14 (App Router)
- <strong>Language</strong>: TypeScript
- <strong>UI Library</strong>: React 18
- <strong>Styling</strong>: Tailwind CSS
- <strong>UI Components</strong>: shadcn/ui, Radix UI
- <strong>State Management</strong>: Recoil

### <strong>Backend & Services</strong>

- <strong>API Routes</strong>: Next.js API Routes
- <strong>Maps</strong>: Google Maps JavaScript API, @react-google-maps/api
- <strong>Places</strong>: Google Places API, react-google-places-autocomplete
- <strong>Payment</strong>: Stripe
- <strong>Authentication</strong>: Clerk

### <strong>Development Tools</strong>

- <strong>Package Manager</strong>: npm
- <strong>Linting</strong>: ESLint
- <strong>Deployment</strong>: Vercel

## <strong>🚀 시작하기</strong>


---


### <strong>사전 요구사항</strong>

- Node.js 18.0 이상
- npm 또는 yarn
- Google Maps API Key
- Stripe API Key
- Clerk API Key

### <strong>설치 및 실행</strong>

1. <strong>레포지토리 클론</strong>

```plain text
git clone https://github.com/yourusername/uber-eats-clone.git
cd uber-eats-clone
```

1. <strong>의존성 설치</strong>

```plain text
npm install
```

1. <strong>환경 변수 설정</strong>

<code>.env.local</code> 파일을 생성하고 다음 환경 변수를 추가합니다:


```plain text
# Google Maps
NEXT_PUBLIC_GOOGLE_API_KEY=your_google_maps_api_key
NEXT_PUBLIC_GOOGLE_MAP_API=your_google_map_id

# Stripe
NEXT_PUBLIC_STRIPE_PUBLISHER_KEY=your_stripe_publishable_key
STRIPE_SECRET_KEY=your_stripe_secret_key

# Clerk
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=your_clerk_publishable_key
CLERK_SECRET_KEY=your_clerk_secret_key
```

1. <strong>개발 서버 실행</strong>

```plain text
npm run dev
```

1. <strong>브라우저에서 확인</strong>

```plain text
http://localhost:3000
```


## <strong>💡 핵심 구현 내용</strong>


---


### <strong>1. Google Maps API 통합</strong>


<strong>GoogleMapSection 컴포넌트</strong>에서 복잡한 지도 기능을 구현했습니다:


```plain text
// 출발지/도착지 상태 변경 시 자동으로 지도 중심 이동
useEffect(() => {
  if (!isEmptyObj(source) && map) {
    map.panTo({ lat: source.lat, lng: source.lng });
    setCenter({ lat: source.lat, lng: source.lng });
  }
}, [source]);

// DirectionsRenderer를 통한 경로 표시
<DirectionsRenderer
  directions={directionRoutePoints}
  options={{
    polylineOptions: {
      strokeColor: "#393938",
      strokeWeight: 10,
    },
    suppressMarkers: true,
  }}
/>
```


### <strong>2. 상태 관리 (Recoil)</strong>


전역 상태를 Recoil atom으로 관리하여 컴포넌트 간 데이터 공유:

- <code>sourceState</code>: 출발지 정보
- <code>destinationState</code>: 도착지 정보
- <code>directionRoutePoints</code>: 경로 정보
- <code>centerState</code>: 지도 중심 좌표

### <strong>3. Stripe 결제 통합</strong>


API Route를 통한 안전한 결제 인텐트 생성:


```plain text
// app/api/create-intent/route.ts
const paymentIntent = await stripe.paymentIntents.create({
  amount: amount * 100, // 센트 단위로 변환
  currency: 'krw',
});
```


## <strong>🔧 트러블슈팅 및 해결 과제</strong>


---


### <strong>1. 한국 환경에서의 Google Maps API 제약</strong>


<strong>문제</strong>: Google Maps의 일부 기능(Transit Directions 등)이 한국에서 제한적으로 지원됨


<strong>해결</strong>:

- Places API와 Directions API를 활용하여 기본 경로 계산 기능 구현
- 커스텀 마커와 오버레이를 사용하여 사용자 경험 개선

### <strong>2. 비동기 지도 로딩 처리</strong>


<strong>문제</strong>: Google Maps API 로딩 전에 컴포넌트가 렌더링되어 에러 발생


<strong>해결</strong>:

- <code>LoadScript</code> 컴포넌트로 API 로딩 관리
- 조건부 렌더링으로 데이터 존재 여부 확인 후 UI 표시

```plain text
{!isEmptyObj(source) && !isEmptyObj(destination) && (
  <DirectionsRenderer directions={directionRoutePoints} />
)}
```


### <strong>3. 타입 안정성 확보</strong>


<strong>문제</strong>: Google Maps API의 복잡한 타입 처리


<strong>해결</strong>:

- TypeScript를 활용한 타입 안전성 확보
- <code>@react-google-maps/api</code>의 타입 정의 활용
