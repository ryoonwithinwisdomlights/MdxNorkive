## <strong>Overview</strong>


---


Norkive is a modern knowledge management platform that bridges the gap between Notion's intuitive editing experience and a high-performance static blog. It automatically converts Notion databases into type-safe MDX content, optimizes images through Cloudinary CDN, and deploys as a blazing-fast Next.js application.


## <strong>Key Features</strong>

- 🔄 <strong>Automated Pipeline</strong>: Notion → MDX conversion with full metadata preservation
- 🖼️ <strong>Image Optimization</strong>: Cloudinary integration with 70% size reduction
- 🎨 <strong>MDX Components</strong>: Rich interactive components with Shiki syntax highlighting
- 🔍 <strong>Advanced Search</strong>: Command palette (<code>⌘K</code>) with fuzzy search
- ⚡ <strong>Performance</strong>: Lighthouse 96/100, under 1s initial load
- 🎯 <strong>Type Safety</strong>: Zod schemas + Content Collections for runtime validation
- 📱 <strong>Responsive</strong>: Mobile-first design with dark mode support

## <strong>Tech Stack</strong>

- <strong>Framework</strong>: Next.js 15 (App Router), React 19
- <strong>Language</strong>: TypeScript 5.6 (87.3% of codebase)
- <strong>Styling</strong>: Tailwind CSS 4.1
- <strong>Content</strong>: MDX + Content Collections + Fumadocs
- <strong>Infrastructure</strong>: Vercel, Cloudinary, Upstash Redis
- <strong>Quality</strong>: ESLint, Prettier, Zod validation

## <strong>Quick Start</strong>


---


### <strong>Prerequisites</strong>

- Node.js >= 20.17.0
- npm/pnpm/yarn

### <strong>Installation</strong>


```plain text
# Clone the repository
git clone https://github.com/ryoonwithinwisdomlights/norkive.git
cd norkive

# Install dependencies
npm install
# or
pnpm install

# Set up environment variables
cp .env.example .env.local

# Run development server
npm run dev
```


Visit [http://localhost:3000](http://localhost:3000/)


### <strong>Environment Variables</strong>


```plain text
# Notion API (optional - for MDX conversion)
NOTION_API_KEY=your_notion_integration_token
NOTION_DATABASE_ID=your_database_id

# Cloudinary (for image optimization)
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret

# Upstash Redis (for caching)
UPSTASH_REDIS_REST_URL=your_redis_url
UPSTASH_REDIS_REST_TOKEN=your_redis_token

# Site configuration
NEXT_PUBLIC_SITE_URL=https://your-domain.com
NEXT_PUBLIC_LANG=kr-KR
```


### <strong>Usage</strong>


---


### <strong>Adding Content</strong>


<strong>Option A: Using Notion (Recommended)</strong>

1. Create content in your Notion database
2. Run the conversion script:

    ```plain text
    npx tsx scripts/notion-mdx-all-in-one.ts
    ```

3. MDX files are automatically generated in <code>content/</code> directory

<strong>Option B: Direct MDX</strong>


Create a file in <code>content/records/my-post.mdx</code>:


```plain text
---
notionId: "unique-id"title: "My Post Title"date: 2025-01-15
category: "Engineering"tags: ["Next.js", "TypeScript"]
---

#Your content here

This is a paragraph with **bold** and *italic* text.

\`\`\`typescript
const example = "code block";
\`\`\`
```

## <strong>Key Technical Decisions</strong>


---


### <strong>Why MDX over Direct Notion Rendering?</strong>

- <strong>Performance</strong>: Static generation vs. runtime API calls (60% faster)
- <strong>SEO</strong>: Complete HTML for crawlers (100% indexing)
- <strong>Customization</strong>: Full control over React components
- <strong>Reliability</strong>: No dependency on Notion API availability

See <FileWrapper names={"ARCHITECTURE.md"} urls={"https://github.com/ryoonwithinwisdomlights/MdxNorkive/blob/main/docs/ARCHITECTURE.md"} /> for detailed analysis.


### <strong>Why Content Collections + Fumadocs?</strong>

- <strong>Type Safety</strong>: Zod schemas prevent runtime errors
- <strong>DX</strong>: Auto-completion and type inference
- <strong>Build-time Validation</strong>: Catch errors before deployment
