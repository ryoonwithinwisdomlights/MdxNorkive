# Security Policy

We take security seriously. If you discover a vulnerability, please follow the steps below.

## Reporting a Vulnerability

Please do NOT open a public issue for security reports.
Instead, email us at: devseryun@gmail.com

Include as much information as possible:

- A clear description of the issue
- Steps to reproduce
- Impact assessment
- Any relevant logs, screenshots, or proof-of-concept details

We will acknowledge your report within 72 hours and provide a timeline for remediation.

## Supported Versions

We support the latest version on the `main` branch.

## Scope

The following areas are in scope:

- Notion API access tokens and database IDs
- Cloudinary API keys and upload configuration
- Upstash Redis credentials
- Build and deployment pipelines

## Responsible Disclosure

Please give us reasonable time to investigate and fix the issue before any public disclosure.
We appreciate your efforts to keep the project and its users safe.
